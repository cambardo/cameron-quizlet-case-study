What we shipped
A new data layer — Can-Do statements
We needed to build an assessment and progress tracking system for our app that felt conversation-first.
What we did
Used LLMs to extract communicative Can-Do statements from 1,000+ lessons and map them to CEFR levels. Repurposed end-of-unit AI roleplays as assessments, then surfaced completed skills in a Skills Hub.

What the data showed
Two failure modes
After launch, the data revealed two structural problems.
Failure mode 1 — conversation drift
AI conversations drifted — taking users down paths where later Can-Dos never surfaced. Users failed objectives not because they lacked the skill, but because the format never gave them the chance to show it.

Failure mode 2 — the bar was too low
Users could satisfy a Can-Do with "yes" or "okay." The system marked objectives complete without verifying the learner used the language we taught. We were measuring participation, not mastery.
Example
AI: "Have you ever been to Japan?" → User: "Yes." ✓ Can-Do marked complete
Target: "I've been to Japan twice — it was amazing."
The deeper problem
We were assessing the wrong thing
Both failure modes share the same root cause.
The root cause
The assessment was built on Can-Do statements. But the curriculum is built on sentence patterns and vocabulary. Can-Dos are a communicative overlay — useful for progress tracking, but not what was actually taught. We were assessing the framing, not the content.

Also: too human-intensive to iterate
V1 required the content team to manually write and QA every assessment. The data is already showing we need to change it — and we can't afford to rebuild by hand every time.
What we need
An assessment generated automatically from the curriculum — grounded in what was actually taught, minimal human authoring, scalable across courses and markets.
The proposal
A generated, frequency-driven assessment
Replace human-authored Can-Do roleplays with a system generated directly from lesson content.
Two inputs from the curriculum
Lesson syntaxes
Structured lesson transcripts the LLM scrapes to rank sentence patterns and vocabulary by frequency across a unit. Highest frequency → what gets assessed.
Grammar groupings
Metadata that clusters pattern variants under a shared grammar concept — so frequency is counted per concept, not per individual pattern.

Key data model change
Grammar concept grouping
The missing metadata that makes frequency-driven assessment generation work.
The problem
A unit may have four patterns all teaching present perfect. Without a grammar concept field, the system treats them as four separate unrelated items — fragmenting the frequency signal. Adding one field links them together.
Present perfect
4 patterns → 1 concept
Have you ever been to [place]?
Have you ever tried [food]?
I've been to [place] before.
I've never tried [food].
Today — no grammar field
{ "pattern_id": "A2_U3_P04", "template": "Have you ever been to [place]?", "unit_id": "A2_U3", "vocab_items": ["Canada", "Japan"] }
Proposed — grammar concept added
{ "pattern_id": "A2_U3_P04", "template": "Have you ever been to [place]?", "unit_id": "A2_U3", "vocab_items": ["Canada", "Japan"], "grammar_concept": "present_perfect", "concept_group_id": "A2_U3_GC_01" }
What this enables
The assessment generator identifies top grammar concepts per unit by frequency, generates exercises for each, and does it without human authoring.
Metrics
How we'd measure success
One data model gap. Two dimensions of impact.
User metrics
Can-Do completion rate — parity across a unit
Baseline exists today — sharp drop-off after Can-Do 2. Target: parity across all Can-Dos in a unit.
Can-Do pass rate after takeaway lesson
Today, takeaway lessons target any grammar error — not the specific pattern being assessed. With grammar concept tagging, practice targets the exact concept the user struggled with. Do users pass on reattempt at a higher rate?
Curriculum efficacy signal
Consistent failures on specific concepts across cohorts become a diagnostic for the curriculum. Closes a feedback loop that didn't exist before.
Business metrics
B2B deal conversion and retention
S4B partners need granular skill data to justify investment. We are actively losing deals today due to weak assessment output.
Content production cost per course
Automated generation eliminates manual authoring and QA — critical as we scale across courses and markets.
DAU / MAU retention
Better assessments give users a clearer sense of progress, reducing churn. Visible skill growth — grounded in what was actually taught — keeps users coming back.
The thread
One missing field — grammar concept grouping — is blocking better user outcomes, a scalable content operation, and B2B revenue. One schema addition. Three unlocks.
Risks & tradeoffs
What could go wrong
01
LLM extraction accuracy
Grammar concept tagging will have errors — especially on ambiguous patterns that could teach multiple concepts. Mitigation: human validation layer with a confidence threshold, below which patterns are flagged for review rather than auto-tagged.
02
Frequency as a proxy for importance
High frequency doesn't always mean pedagogically significant — a pattern might repeat because it's filler, not a teaching target. Mitigation: frequency is a starting signal, not the final decision. Human curriculum review of top-ranked items before assessment generation.
03
Grammar grouping edge cases
Some patterns span multiple grammar concepts. Forcing a single concept group could misrepresent what's being taught. Mitigation: allow multiple concept tags per pattern, with a primary tag designated for frequency counting.
